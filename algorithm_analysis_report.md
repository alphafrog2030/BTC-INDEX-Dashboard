# 비트코인 온체인 인덱스 분석 보고서 (AI 및 개발 참고 목적)

본 문서는 BTC 온체인 인덱스의 투자 지표, 점수화 로직 및 시뮬레이터(미래 예측) 알고리즘에 대한 분석 보고서입니다. 향후 알고리즘 개선 및 다른 AI 에이전트의 이해를 돕기 위해 작성되었습니다.

## 1. 점수화 인디케이터 (투자 참고 지표)
현재 시스템은 프론트엔드(`marketService.ts`)에서 백엔드 데이터를 수집하여 6개의 주요 지표를 기반으로 시장을 평가하고 있으며, 지표별로 가중치가 다르게 설정되어 있습니다.

| 지표명 (Indicator) | 가중치 | 데이터 출처 및 속성 |
|-------------------|-------|------------------|
| **MVRV Z-Score**  | 27.5% | 온체인 (과대/과소평가 측정) |
| **Puell Multiple**| 17.5% | 온체인 (채굴자 수익성) |
| **NUPL**          | 17.5% | 온체인 (순 미실현 수익/손실) |
| **200 Week MA**   | 17.5% | 시장 데이터 (장기 이동평균선 이격도) |
| **Fear & Greed**  | 12.5% | 심리 지표 (공포 탐욕 지수) |
| **Funding Rate**  | 7.5%  | 파생상품 (선물 펀딩비) |

> **참고**: `Reserve Risk`와 `SOPR` 지표 또한 점수화 로직(`calculateScore`)에는 정의되어 있으나, 현재 프론트엔드의 최종 종합 점수 산출(`rawIndicators` 파트)에는 포함되지 않아 실제 가중치가 반영되지 않습니다.

## 2. 지표 기반 점수화 로직 (Scoring Logic)
각 지표는 현재 값에 따라 0점(SELL, 매도)부터 10점(BUY, 매수)까지의 개별 점수(`score`)를 부여받습니다. 결측치가 있는 지표의 경우 기본값인 **5점(NEUTRAL)**이 부여됩니다. 이후 각 지표의 가중치를 곱하여 `종합 점수(0~100점)`를 산출합니다. 

### 2.1. 개별 지표별 점수 할당 로직

* **MVRV Z-Score (과열 점수)**
  * `<= 0.0` : 10점 (BUY)
  * `<= 1.0` : 8점 (BUY)
  * `>= 3.0` : 2점 (SELL)
  * `>= 7.0` : 0점 (SELL)

* **Puell Multiple (채굴자 항복 점수)**
  * `<= 0.5` : 10점 (BUY)
  * `<= 1.0` : 8점 (BUY)
  * `>= 2.5` : 2점 (SELL)
  * `>= 4.0` : 0점 (SELL)

* **NUPL (공포 편향 점수)**
  * `< 0.0` : 10점 (BUY)
  * `<= 0.25`: 8점 (BUY)
  * `>= 0.60`: 2점 (SELL)
  * `>= 0.75`: 0점 (SELL)

* **200 Week MA 비율 (현재가 / 200주 이평가 비율)**
  * 비율 `<= 1.0` : 10점 (BUY)
  * 비율 `<= 1.5` : 8점 (BUY)
  * 비율 `>= 2.5` : 2점 (SELL)
  * 비율 `>= 5.0` : 0점 (SELL)

* **Fear & Greed (공포 탐욕 점수)**
  * `<= 20` : 10점 (BUY)
  * `<= 40` : 8점 (BUY)
  * `>= 75` : 2점 (SELL)
  * `>= 80` : 0점 (SELL)

* **Funding Rate (롱/숏 펀딩 비율)**
  * `<= -0.01` : 10점 (BUY)
  * `<= 0.01` : 8점 (BUY)
  * `>= 0.05` : 2점 (SELL)
  * `>= 0.10` : 0점 (SELL)

*(해당 구간 이외의 모호한 값은 모두 안전값인 5점으로 평가)*

### 2.2. 최종 합산 및 해석 로직 (Weighted Total)
1. **각 지표별 가중치 적용**: `Sum(개별 지표 점수 * 개별 지표 가중치) / 100` 계산을 통해 예비 총 합산 도출 (10점 만점 형태)
2. **100점 분률 조정**: `최종 종합 점수 = Min(Round(위 합산값 * 10), 100)`
3. **투자 자문 텍스트 생성 (`generateKoreanAnalysis`)**:
   * **90점 이상**: 💎 강력한 매수 신호 (Strong Buy)
   * **70점 ~ 89점**: ✅ 매수 우위 (Accumulate) - 적극적인 적립 및 매수 추천 구간
   * **31점 ~ 69점**: ⚖️ 중립 (Neutral) - 관망 및 소규모 분할매수
   * **30점 이하**: ⚠️ 과열 경보 (Overheated) - 이익 실현 및 보유 물량 매도 고려 구간

## 3. 시뮬레이터 (미래 가격 예측) 알고리즘 (`Simulator.tsx` / `historicalData.ts`)
과거 12년간의 데이터(주 단위 프랙탈 포인트)를 스캔하여 가동되며 현재와 가장 유사한 **"쌍둥이 구간(Fractal)" TOP 3**를 도출해 미래 배수를 산정합니다.

### 3.1. 과거 패턴 매칭 알고리즘 (`calculateSimilarity`)
입력된 3가지 지표(`MVRV Z-Score`, `200W MA Ratio`, `MVRV 60일 추세 방향`) + `반감기 주기 단계`를 기준으로 유사도(0 ~ 100%)를 산출합니다.

1. **단기 추세 필터 (Trend Matching)**: 
   * 현재의 60일 추세(`60d Slope`, Uptrend/Downtrend) 방향이 과거 구간과 일치하지 않으면 즉시 **유사도를 0**으로 처리하고 탈락시킵니다.
2. **지표 오차율 기반 일치도 (State Matching - 백분율 퍼센트)**: 
   * 현재 값과 과거 값의 백분율 오차를 산정하여 `weightedAvgPercentDiff`를 계산합니다. 
   * **가중치 부여**: `MVRV Z-Score (70%)`, `200W MA 비율 (30%)`
3. **반감기 사이클 페널티 (Halving Cycle Filter)**: 
   * 두 시점 간 '마지막 반감기로부터 경과된 일수(`Days since Last Halving`)' 차이를 계산합니다.
   * 사이클 템포가 달라질 경우, 그에 비례해 최대 **30%까지 점수를 감산**합니다. (`halvingPenalty = Min(30, (일수 차이 / 365) * 15)`)
4. **쌍둥이 시점(Fractal Point) 최종 선정**: 
   * 95% 이상 일치하는 지점들은 최신순으로 우선 정렬하고, 나머지는 일치도 순으로 정렬합니다. 
   * 여러 개의 후보군 중 동일한 추세 편향을 피하기 위해 최상위 3개의 포인트는 **서로 최소 2년 이상 떨어져 있어야** 한다는 쿨타임 제약 조건을 두고 TOP 3 과거 시점을 결정합니다.

### 3.2. 가격 프로젝션 로직 (Price Projection)
결정된 TOP 3 과거 시점으로부터 사용자가 입력한 `N개월` 뒤 역사적 데이터의 실제 투자 성과(배수)를 추적합니다. 
단, 과거의 수백 배에 달하는 비상식적 폭등을 그대로 출력하지 않고, 다음과 같은 **시장 파이(시가총액 규모) 팽창 조정 알고리즘**을 필수로 거칩니다.

* **Logarithmic Diminishing Returns (대수적 수익률 체감법 적용)**
  * 과거 가격($10 등)에 비해 현재 가격($60,000 등)이 훨씬 크면 시장 자본금 크기로 인해 상승률이 낮아지는 현상을 반영합니다.
  * 둔화 계수 공식: `dimFactor = log10(과거 시점 가격) / log10(현재 시점 가격)` 
  * 패널티 적용 하한선을 **최소 0.35**로 방어하여, 너무 극한의 역성장 계산이 나오지 않게 제어합니다. (최대 65%까지만 상승분을 커트합니다.)
* **최종 배수(Multiplier) 연산**
  * 가격 상승 시 (과거 N개월 후 이득 상황): `최종 수익 배수 = 1 + ((역사적 원본 배수 - 1) * dimFactor)`
  * 가격 하락 시 (과거 N개월 후 손실 상황): `최종 수익 배수 = 1 - (하락폭 * dimFactor)`

### 3.3. 시나리오 출력
차트와 수익률을 3가지 시나리오로 제시해 편향 없는 뷰를 제공합니다.
* **Best Case (최상의 시나리오):** 수집된 TOP 3 프로젝션 중 **최댓값**.
* **Most Likely (평균 시나리오):** 수집된 TOP 3 프로젝션의 **평균치** (이 값을 기준선으로 사용).
* **Worst Case (최악의 시나리오):** 수집된 TOP 3 프로젝션 중 **최솟값**.

## 4. 로직 보완을 위한 추후 접근 권장 사항 (To-Do for AI)
코드 분석 결과 하드코딩된 부분과 확장성이 닫혀 있는 부분이 일부 존재하여 향후 고도화 여지가 보입니다:
1. `시장 예측 Simulator`에서 사용하는 지표에 `Puell Multiple`과 `NUPL` 등을 확장하여 유사도 판별의 정밀도를 올리기 (`historicalData.ts` 수정 필요).
2. `marketService.ts`의 채택되지 않은 유휴 지표(`Reserve Risk`, `SOPR`)를 점수화 가중치에 마저 정식 편입하거나 걷어내기.
3. Diminishing Factor(둔화 계수)에 가격 로그값을 취하는 것뿐만 아니라, **실제 온체인 시가총액/M2 통화량 지표를 혼합**해 계산하도록 발전 가능성 염두.
