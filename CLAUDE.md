# CLAUDE.md — BTC On-Chain Index Dashboard

## 프로젝트 개요
비트코인 온체인 지표를 자동 수집·점수화하여 투자 진입 시점을 쉽게 판단할 수 있게 돕는
대시보드 서비스. 복잡한 온체인 개념을 모르는 **초보 크립토 투자자**가 직관적으로
"지금 사도 되는가?"를 확인할 수 있도록 설계된다.

퍼블릭 웹사이트 → 모바일 앱 순으로 런칭하여, 광고 및 유료 구독 서비스로 수익화한다.

---

## 핵심 개발 원칙: 실전 투자 효용성 최우선

> **이 서비스의 생명은 신뢰도다.**
> 알고리즘과 로직은 항상 "실제 크립토 투자 전문가가 자문한다면 이 로직을 쓰겠는가?"
> 라는 기준으로 평가해야 한다. 시각적으로 멋지거나 기능이 많더라도
> 투자 효용성이 낮으면 의미 없다.

### 알고리즘 작업 시 체크 기준
- 지표 가중치·임계값을 변경할 때는 반드시 실제 BTC 시장 사이클
  (2018 바닥, 2020 반감기, 2021 고점, 2022 바닥 등) 기준으로 백테스트 결과를 확인하고 조정한다
- 시뮬레이터의 유사 구간 탐색 로직은 표면적 수치 유사성이 아니라
  **시장 사이클 맥락**(바닥/상승/과열/하락)이 유사한 구간을 찾도록 설계한다
- 점수 0~100의 의미가 실제 시장 국면과 일치하는지 주기적으로 검증한다
- 새 지표 추가 시 "이 지표가 실제 투자 타이밍에 유의미한가"를 먼저 검토하고 추가한다

---

## 타깃 사용자
- **초보 크립토 투자자** — 온체인 지표 개념을 모르지만 비트코인에 투자하고 싶은 사람
- 핵심 니즈: "지금 사도 돼?" → 복잡한 설명 없이 점수 하나로 판단하고 싶음
- UX 원칙: 전문 용어 최소화, 숫자보다 신호(BUY/HOLD/SELL)와 시각적 게이지 우선

---

## 비즈니스 목표
- 1단계: 퍼블릭 웹사이트 런칭 → 광고 수익 (애드센스 등)
- 2단계: 유료 구독 — 프리미엄 알림(진입 시점 푸시), 상세 리포트, 시뮬레이터 고급 기능
- 3단계: 모바일 앱(iOS/Android) 확장

---

## 코딩 원칙
- 파일 수정 전 반드시 해당 파일을 먼저 읽고 기존 로직 파악 후 수정
- 보안: API 키는 환경변수/설정 파일로만 관리, 프론트엔드 노출 절대 금지
- 불필요한 의존성 추가 금지, 컴포넌트 단순화 우선
- 히스토리컬 데이터 로딩 성능을 항상 의식하고 최적화한다
